# パーミッション

Claude Codeが実行できる操作の制御方法を解説します。

## パーミッションの概念

Claude Codeは以下のカテゴリの操作を実行できます:

| カテゴリ | 操作例 |
|---|---|
| 読み取り | ファイル読み込み、検索、glob |
| 書き込み | ファイル作成・編集・削除 |
| 実行 | Bashコマンド、npm run、テスト実行 |
| ネットワーク | Web検索、API呼び出し |

## デフォルトの動作

デフォルトでは、Claude Codeは操作前にユーザーの確認を求めます。

```
◆ ファイルを編集しますか？
  src/api/users.py
  [y/n/a/d]
```

## 自動承認の設定

`.claude/settings.json` で自動承認ルールを設定できます。

```json
{
  "permissions": {
    "allow": [
      "Read(*)",
      "Glob(*)",
      "Grep(*)",
      "Bash(npm run test)",
      "Bash(npm run lint)"
    ],
    "deny": [
      "Bash(rm -rf*)",
      "Bash(git push --force*)"
    ]
  }
}
```

## ツール別パーミッション

### 読み取り系（低リスク）

- `Read` — ファイル読み込み
- `Glob` — ファイルパターン検索
- `Grep` — 内容検索

### 書き込み系（要確認）

- `Edit` — ファイル編集
- `Write` — ファイル作成
- `Bash` — シェルコマンド実行

## セーフティガード

以下の操作は、明示的な許可がない限り実行されません:

- `git push --force` — 強制プッシュ
- `rm -rf` — 再帰的削除
- 本番環境への変更
- 機密ファイル（.env等）の変更

## --dangerously-skip-permissions

⚠️ **注意**: このオプションは全ての確認をスキップします。

```bash
claude --dangerously-skip-permissions
```

自動化スクリプトや信頼できる環境でのみ使用してください。
本番環境や重要なリポジトリでは絶対に使用しないでください。

## パーミッションのベストプラクティス

1. **最小権限の原則** — 必要なツールのみ許可する
2. **読み取り専用モード** — 調査・説明タスクには Read, Glob, Grep のみ許可
3. **テスト環境での使用** — 本番環境へのアクセスは制限する
4. **定期的な見直し** — settings.json の設定を定期的に確認する
