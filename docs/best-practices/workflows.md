# ワークフロー

Claude Codeを使った効率的な開発ワークフローを紹介します。

## 機能開発フロー

### 1. 仕様の確認・作成

```
> /spec
```

既存コードを分析して仕様書を生成します。

### 2. 実装計画の立案

```
> /plan ユーザー通知機能を追加したい
```

プランモードで実装方針を確認してから着手します。

### 3. 段階的な実装

```
> まずデータモデルだけ実装してください
> 次にサービス層を実装してください
> 最後にAPIエンドポイントを追加してください
```

### 4. テストの追加

```
> 実装した機能のテストを追加してください
```

### 5. コードレビュー

```
> /review
```

## バグ修正フロー

```bash
# エラーメッセージをそのまま貼り付ける
> TypeError: Cannot read property 'id' of undefined
  at UserCard.tsx:42

# または
> npm test を実行したら以下のエラーが出ました:
  [エラー内容をペースト]
```

## リファクタリングフロー

大規模なリファクタリングは段階的に行います。

```
> /plan src/utils.ts を機能別にファイル分割したい

> ステップ1: formatDate 関連の関数を src/utils/date.ts に移動
> ステップ2: string 操作の関数を src/utils/string.ts に移動
> ステップ3: 既存のimportを更新
```

## コードレビュー支援

PRを作る前にClaude Codeでセルフレビューします。

```
> git diff main...HEAD の変更をレビューしてください。
  セキュリティ問題、パフォーマンス問題、バグの可能性を指摘してください。
```

## ドキュメント生成

```
> src/api/ ディレクトリのAPIドキュメントをOpenAPI形式で生成してください
> 変更履歴をCHANGELOG.md に追記してください
```

## CI/CD との連携

```bash
# コミット前にコードレビュー（Git Hook）
#!/bin/bash
claude "/review staged" --output-format=text
```

## チーム開発でのTips

- CLAUDE.md でチーム全員の規約を統一する
- カスタムコマンドをチームで共有する
- プランモードで設計をレビューしてからコーディングする
