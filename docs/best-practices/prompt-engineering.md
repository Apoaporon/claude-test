# プロンプトエンジニアリング

Claude Codeから最高の結果を引き出すためのプロンプト設計のコツを解説します。

## 基本原則

### 具体的に指示する

❌ 悪い例:
```
> コードを直してください
```

✅ 良い例:
```
> src/api/users.py の getUserById 関数で、ユーザーが存在しない場合に
  None ではなく 404 例外を返すように修正してください
```

### コンテキストを提供する

```
> このプロジェクトはFastAPI + PostgreSQLで構築されています。
  src/models.py にPydanticモデルがあります。
  ユーザー登録APIを追加してください。
```

### 期待する出力形式を明示する

```
> READMEを更新してください。
  - 形式: Markdown
  - セクション: インストール、使い方、API仕様、コントリビュート
  - 長さ: 200-300行程度
```

## 効果的なパターン

### ステップバイステップ指示

```
> 以下の手順でOAuth2認証を実装してください:
  1. google-auth ライブラリをインストール
  2. 環境変数 GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET を追加
  3. /auth/google エンドポイントを作成
  4. コールバックハンドラを実装
```

### 制約を明示する

```
> パスワードハッシュ化を実装してください。
  - bcrypt を使用（argon2は使わない）
  - 既存のUserモデルは変更しない
  - テストも追加する
```

### 参照ファイルを指定する

```
> @src/models.py と @src/routers/users.py を参考に、
  Productモデルと対応するCRUD APIを作成してください。
```

## CLAUDE.md を活用する

プロジェクト固有のルールをCLAUDE.mdに書いておくことで、
毎回指示しなくても規約に沿ったコードが生成されます。

```markdown
## コーディング規約
- 型ヒント必須
- 関数は20行以内
- テストは pytest を使用
- コメントは日本語で書く
```

## よくある失敗パターン

### 曖昧な指示

```
> いい感じにしてください  ← 何を求めているか不明
> バグを直して           ← どのバグか不明
> リファクタリングして   ← 方針が不明
```

### 過度に大きなタスク

一度に多くの変更を依頼すると、エラーのリスクが高まります。
タスクを小さく分割して、段階的に実装しましょう。
