# プランモード

プランモードは、コードを変更する前に実装計画を立案・確認するための機能です。

## プランモードとは

通常モードでは、Claudeは即座にコードを編集します。
プランモードでは、まず実装方針を提示し、ユーザーの承認後に実装を開始します。

## 起動方法

### コマンドラインから

```bash
claude --plan "新しいAPIエンドポイントを追加したい"
```

### インタラクティブモードから

```
> /plan
```

### スラッシュコマンドから

```
> /plan 認証機能を実装してください
```

## プランの内容

プランモードでは以下が提示されます:

1. **実装方針** — どのようなアプローチを取るか
2. **変更予定ファイル** — 編集・作成するファイルの一覧
3. **実装ステップ** — 順序立てた手順
4. **考慮事項** — トレードオフや注意点

## 使用例

```
> /plan ユーザー認証をJWTで実装してください

Claude: 以下の実装計画を提案します：

## 実装方針
JWTベースの認証を実装します。

## 変更予定ファイル
- src/auth/jwt.py（新規作成）
- src/routers/auth.py（新規作成）
- src/models.py（User モデル追加）
- main.py（ルーター追加）

## 実装ステップ
1. Pydanticモデルにユーザーモデルを追加
2. JWT生成・検証ロジックを実装
3. ログイン・登録エンドポイントを作成
4. 認証ミドルウェアを実装

この計画で進めますか？
```

## プランモードのメリット

- 大規模な変更前に全体像を把握できる
- 不要な変更を事前に除外できる
- チームでの方針共有に使える
- 複雑なリファクタリングの段取りを確認できる

## プランの修正

プラン提示後、修正を依頼できます:

```
> テストの追加も含めてください
> データベースはSQLiteではなくPostgreSQLを使ってください
```
